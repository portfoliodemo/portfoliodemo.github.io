Web Layouts Using CSS Grid

CSS Grid - What is it? CSS Grid is an immensely powerful web layout tool that opens up a whole new world of design possibilities, taking us from the mundane to the magnificent with only a few simple lines of code.

Through the power of CSS Grid we can define our own custom column and row structures, expanding or contracting them as we see fit, while also providing the opportunities to tailor grid lines, and specify individual cells within the grid as unique grid areas too.

All of these features within compact properties that are independently accessible and customizable, leading the way to a near infinite amount of choices for our layouts and web designs.

Giddy About Grid

All joking aside, CSS Grid is a tool that can unleash all sorts of unique customizations, so much so that I almost don't know where to start when trying to talk about it! You know when your binging on development docs your excited.

So what kind of setup is required when you want to start making and using a grid, or in many cases, grids? Well it's pretty simple to get up and running, you just select virtually any element from your html that you want to become a grid, and you set it's display property to grid, as in this CSS example shown below:

.our-awesome-grid { display: grid; }

Okay cool! So we now have an element declared as a grid, and our element belongs to the class called our-awesome-grid, which we could have named pretty much anything we want. In this case, we are going to use our grid as a container to play around with some of the many properties of grid!

A key point to remember is that only the direct/first-level descendant(s) of an element with display: grid will inherently become a grid item or items, so it's important to keep those relationships in mind as you are developing your site and building out your grids. As we dig a little deeper hopefully this will become clearer. Now that we have our grid defined let's brainstorm some layout ideas!

While a one column layout may be just what you need, and that's effectively all that's created when you initially declare an element as a grid (items will still be laid out according to normal flow and still look very similar if not the same as before declaring a grid), for demonstration sake that's not exactly ideal. So, let's go ahead and start off our demo by creating a let's say, hmm, pick a number? Alright a three-column grid it is then!

From Grid to Great

We can create a basic three column grid by simply utilizing a property called grid-template-columns, and specifying a value containing any length measurement available, such as pixels (px), ems (em), percentages (%), fractions (fr), whatever suits your situation at any given moment.

In this case we will be using fr, which stands for fraction and assigns all of the available space to that portion of the grid based on the fraction. Here's an example of what CSS might be used to setup a basic three column layout:

.our-awesome-grid { display: grid; grid-template-columns: 1fr 2fr 1fr; }

Alright! Now we are definitely gettin' griddy with it. We've declared our grid container, and defined a three-column layout within it, with the middle column taking up half of the available space.

Column to an End

Our basic grid has been setup and we now have three columns we can play around with. We could have just as easily setup our three columns to become two, or five, or twelve columns, all really up to you to use however you want, demonstrating the flexibility of not only grid in general, but also of its many specific properties as well!

So again, using only grid-template-columns, we could also write something like:

.our-awesome-grid { display: grid; grid-template-columns: 200px 1fr; }

or:

.our-awesome-grid { display: grid; grid-template-columns: repeat(12, 1fr); }

Whoa what's up with the repeat function lookin' thingy?! Twelve comma one fr? What on Mars is that supposed to mean!? Wait, did you say..Mars? Awesome.

So, you're telling me your just going to lazily create twelve repeating columns, specifying each one to take up just one fraction of the available space, just like that? Sweet!! If you'd like to check out some more info on CSS functions then you can head over to the MDN Web Docs site and read up all about them here.

Whether or not you need a gazillion columns or rows will ultimately be up to you and your design specs, but perhaps more often than not sticking with a simple column and row structure, will in turn yield excellent results.

We are cruising along now, we have some code snippets to build on and we are ready to move on to rows! So, you are this far into the project, potentially over your limit, and you're telling me all you have really mentioned so far is pretty much grid and grid-template-columns? Yes, yes I am.

Row, Row, Row Your Grid!

Forget columns! It's time to Rock n' Row! Apologies to my classmates (and instructor) of INTP-362 who have to read this!! Thank you for your help and patience! =)

To start placing rows into our demo we are going to take advantage of grid-template-rows. Similar to grid-template columns but now we are essentially placing rows along the 'y-axis' if you will, whereas with columns we were working across the 'x-axis'. While we're on the subject, we can also utilize the z-index property to stack items however we see fit on the z-axis as well! Again, CSS Grid opens up a near infinite amount of design possibilities as we are essentially able to create in 3D! But for now, back to rows!

If we were trying to create three rows we could set something up very similar to our column code, only this time it could look like:

.our-awesome-grid { display: grid; grid-template-rows: 1fr 2fr 1fr; }

While the syntax is very similar to grid-template-columns, there are some subtle differences that have to be taken into account when differentiating between the two. Wait, is there? Are you sure? No I'm not actually, not at all, upon closer inspection they look almost identical if I'm not mistaken?

We can definitely combine columns and rows together to help define our layouts even more. So, combining our code examples from earlier, we could set up something like the following:

.our-awesome-grid { display: grid; grid-template-columns: 1fr 2fr 1fr; grid-template-rows: 1fr 2fr 1fr; }

Or perhaps we want five rows instead of three, again not much is needed to set that up, setting the values as we see fit such as:

.our-awesome-grid { display: grid; grid-template-columns: 1fr 2fr 1fr; grid-template-rows: 100px 2fr 4fr 10em 1fr; }

Yeah, now that you mention it, grid-template-columns and grid-template-rows are pretty similar, which leads me to wonder, how did they really build the pyramids? Or perhaps an even more important question to ask, is why did they build the Great Pyramid of Giza?

A Hieroglyphic-less Tomb?

General consensus seems to be that the pyramids were built to be tombs, but a potential wrinkle in that well established theory is that there is almost no hieroglyphics anywhere in the pyramids at Giza, aside from some small markings believed to have been left by workmen in the pyramid of Khufu, known as the Great Pyramid.

Why is the absence of hieroglyphics a noticeable omission from the potentially greatest tomb ever constructed? Roughly 665km south of Giza on the western bank of the Nile opposite Luxor (ancient day Thebes), a vast majority of Pharaohs and royals are buried in a necropolis of elaborate tombs and tunnels known as the Valley of the Kings.

Seemingly nearly every wall within every royal tomb is intricately decorated from floor to ceiling with hieroglyphics and incredible artwork, usually documenting the inevitably successful but perilous journey through the many challenges of the afterlife that the occupier of the tomb must pass, according to ancient Egyptian beliefs. One of the many well-known Pharaohs buried there was a young boy known as Tutankhamun.
