So what kind of setup is required when you want to start making and using a grid, or in many cases, grids? Well it's pretty simple to get up and running, you just select virtually any element from your html that you want to become a grid, and you set it's display property to grid, as in this CSS example shown below:

.my-awesome-site { display: grid; }

Okay cool! So now we now have an element declared as a grid (our element here is a div, but again you can use virtually any html element to become a grid), and our element belongs to the class called my-awesome-site. In this case, we are going to use our grid as a container for the main portion of our site where we play around with some of the properties of Grid such as columns and rows and gaps.

A key point to remember is that only the direct/first-level descendant(s) of an element with display: grid will inherently become a grid item or items. So it's important to keep those relationships in mind as you are developing your site and building out your grids. As we dig a little deeper hopefully this will become clearer. Now is probably a perfect time to brainstorm some web layout demo ideas!

While a one column layout may be just what you need, and that's effectively all that's created when you initally declare an element as a grid (items will still be laid out in normal flow just the same as before declaring a grid), for demonstration sake that's not exactly ideal. So let's go ahead and start off our demo by creating a three column grid!

From Grid to Great

We can create a basic three column grid by simply utilizing a property called grid-template-columns, along with a value containing any length measurement you'd like, pixels (px), ems (em), fractions (fr), whatever suits your situation at any given moment.

In this case we will be using fr, which stands for fraction and assigns all of the available space to that portion of the grid based on the fraction. Here's an example of what CSS might be needed to setup a basic three column layout:

.my-awesome-site { display: grid; grid-template-columns: 1fr 2fr 1fr; }

Alright! Now we are definitely gettin' griddy with it. We've declared our grid container, and defined a three column layout within it, with the middle column taking up half of the available space.

Seeing the CSS in action along with some html markup will really help to showcase what's happening here. So without further ado let's go ahead and setup a skeleton demo and start having even more fun with grid! Queue the montage! (Just kidding, there's no montage, yet..)

Column to an End

Our basic grid has been setup and we now have three columns we can play around with. We could have just as easily setup our three columns to become two, or five, or twelve columns, all really up to you to use however you want, demonstrating the flexibility of not only Grid in general, but also of its many specific properties as well!

So again using only grid-template-columns, we could also write something like:

.my-awesome-site { display: grid; grid-template-columns: 200px 1fr; }

or:

.my-awesome-site { display: grid; grid-template-columns: repeat(12, 1fr); }

Whoa what's up with the repeat function lookin' thingy?! Twelve comma one fr? What on Mars is that supposed to mean!? Wait....Mars? Awesome. So your telling me your just going to lazily create twelve repeating columns, specifiying each one to take up just one fraction of the available space, just like that? Sweet!!

Whether or not you need a gazillion columns or rows will ultimately be up to you, but perhaps more often than not sticking with a simple column and row structure, and nesting grids if needed, will yield excellent results.

Row, Row, Row Your Grid!

Forget columns! It's time to Rock n' Row! Apologies to my classmates (and instructor!) of INTP-362 who have to read this!! =) Thank you for your patience, help, and forgiveness! ;)

To start placing rows into our demo we are going to take advantage of grid-template-rows. Similar to grid-template columns but now we are essentially placing rows along the 'y-axis' if you will, whereas with columns we were working across the 'x-axis'. While we're on the subject, we can also utilize the z-index to stack items however we see fit as well! Opening up another near infinite amount of design possibilites as we are essentially able to create in 3D! But for now, back to rows!

If we were trying to create three rows we could set something up very similar to our column code, only this time it could look like:

.my-awesome-site { display: grid; grid-template-rows: 1fr 2fr 1fr; }

While the syntax is very similar to grid-template-columns, there are some subtle differences that have to be taken into account when differentiating between the two. Wait, is there? Are you sure? No I'm not actually, not at all, upon closer inspection they look almost identical if I'm not mistaken?

We can definitely combine columns and rows together to help define our layouts even more. So combining our code examples from earlier, we could set up something like the following:

.my-awesome-site { display: grid; grid-template-columns: 1fr 2fr 1fr; grid-template-rows: 1fr 2fr 1fr; }

Yeah, now that you mention it, grid-template-columns and grid-template-rows are pretty similar, which leads me to wonder, how did they really build the pyramids? Or perhaps an even more important question to ask, is why did they build the Great Pyramid of Giza?
