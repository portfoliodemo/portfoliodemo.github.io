Setting Up Your Layouts

Wait is that grid above supposed to be something? Never mind about that now. Starting with a simple mobile design layout, and expanding it to larger screens and more complex layouts, is a lot easier than taking a complex website and shrinking it down to a mobile friendly site. Simplicity however, has a tendency to oftentimes become strikingly complex as you begin to strive for it. As the famous writer Mark Twain once said "I didn't have time to write a short letter, so I wrote a long one instead.".

Let's go ahead and build out a simple mobile layout, and see if we can scale it up with relative ease using a few key features of CSS Grid.

.mobile-layout { display: grid; grid-template-columns: 1fr; grid-template-areas: "header" "nav" "main" "aside" "footer"; }

Okay, so we have declared our .mobile-layout class to become a grid, set the column structure to 1fr, and labelled our grid template areas with the names that we'd like to use to identify our core elements. Note we could have named "header", "nav", "main" etc. anything we wanted, "header" could be "masthead" or "Oilers-Rule" if we were so inclined. What's interesting here is that we can easily change the visual representation of our layouts with grid-template-areas and grid-template-columns, without affecting the logical source order of our markup, a huge advantage and key component when it comes to addressing accessibility issues.

Setting up our layout this way is a two step process, first we define the names we wish to use as shown above with grid-template-areas, and then, within each element that we want to include in our grid layout, we set the grid-area property to the name we chose to set. Without sounding more confusing, let's just take a look at that code.

.header-content { grid-area: header; }
.navigation { grid-area: nav; }
.main-content { grid-area: main; }
.aside-content { grid-area: aside; }
.footer-content { grid-area: footer; }

Note that there are no quotations used when setting the grid-area property. Now we have setup our basic mobile layout, and we can easily adapt our visual layout to accommodate varying screen sizes and viewports by adding in some simple media queries that alter the grid-template-columns and grid-template-area properties, and voila, responsive site! Grid vs. Flexbox

What does the power of grid mean for the mighty flexbox? CSS Grid was not meant to replace flexbox, but instead, to work in conjunction together with flexbox to provide even more options to developers. Flexbox is intended for manipulation of content on one axis, while Grids is designed for content manipulation on two. There are infinite examples and opportunities where you may want to use flex within a grid and vice versa as you go about designing your webpages, continually opening up the world of possibilities for developers to create any layout we can imagine!
